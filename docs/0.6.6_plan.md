# v0.6.6 优化计划（对齐 OpenClaw 执行/代码/对话能力）

## 1. 目标与边界

### 1.1 最终目标

对齐 OpenClaw 在以下三类能力上的工程表现：

1. **执行能力**：可持续、可观测、可审批的命令与进程执行
2. **代码能力**：稳定完成代码修改、验证、回归、诊断
3. **对话能力**：上下文一致、人格一致、群聊安全、心跳主动但不过载

### 1.2 当前版本基线

- 基线版本：`v0.6-backup-20260308`（tag）
- 当前状态文档：`/root/swarmbot_dev/spec_0.6.md`

---

## 2. OpenClaw 设计对比结论

基于 `docs/reference` 与 `templates/AGENTS.md` 的可见设计要点，对比当前 SwarmBot：

### 2.1 已对齐项

- 会话启动前读取人格/用户/记忆文件的方向已具备（SOUL 注入已落地）
- 有后台循环与主动检查框架（Overthinking/Overaction）
- 有工具系统与技能注册体系（SkillRegistry + ToolRegistry）

### 2.2 未对齐关键项

1. **工具执行面不足**
   - 缺少 exec/process/PTY 的完整能力分层
   - 缺少强约束审批与危险命令策略
   - 进程管理能力（poll/send-keys/paste/kill）尚未体系化

2. **调度与心跳体系不足**
   - 缺少统一 schedule 模型（at/cron/every）
   - 缺少 delivery 模式（announce/webhook/none）标准化
   - 缺少 wakeMode（now/next-heartbeat）机制

3. **会话规则与共享上下文治理不足**
   - 群聊“何时发言/何时静默”规则不够结构化
   - 主会话与共享会话的数据隔离策略仍需显式化

4. **回归与健康保障不足**
   - 模型不可用时缺少 fail-fast 与结果分级
   - 基准回归指标未形成统一看板

## 3. v0.6.6 分阶段实施计划

## Phase A：执行能力对齐（最高优先级）

### A1 工具能力扩展

- 新增 `swarm_exec`（前台/后台/超时）
- 新增 `swarm_process`（poll/send_keys/paste/kill）
- 新增 `swarm_exec_approval`（危险命令审批）

### A2 安全与参数健壮性

- 统一参数清洗与 schema 校验
- 完善 `ToolRegistry` 非法参数过滤与日志分级
- 高危命令默认 deny，审批后一次性放行

---

## Phase B：调度能力对齐（高优先级）

### B1 Overaction Step 6 调度引擎

- 定义任务结构：`at/cron/every`
- 定义交付结构：`announce/webhook/none`
- 定义唤醒结构：`now/next-heartbeat`

### B2 Overaction Step 7 自我诊断

- 对话性能指标归档（延时、完成率、异常率）
- 模块健康测试（memory/tool/loop）
- 诊断报告写入 Hot Memory

---

## Phase C：外部事件驱动（高优先级）

### C1 Overthinking 外部检查插件化

- 邮件/日历/天气/Git 状态检查器统一接口
- 阈值触发规则可配置

### C2 事件到行动的联动

- `Overthinking -> urgent_event -> Overaction.trigger(events)`
- 处理中断策略：紧急事件可抢占当前周期

---

## Phase D：对话能力与会话治理（中优先级）

### D1 OpenClaw 风格模板体系

- 补齐并串联：`IDENTITY.md / USER.md / HEARTBEAT.md / TOOLS.md`
- 明确主会话与共享会话的读取矩阵

### D2 发言策略治理

- 群聊发言策略：触发发言/静默条件
- 心跳策略：主动检查频率与打扰控制

---

## 4. 数据集回归与验收标准（qwen3-coder-next）

### 4.1 模型与配置

- 主模型：`qwen3-coder-next`
- 回归必须记录：model/base_url/max_tokens/loop_profile

### 4.2 回归集

1. `scripts/eval_logic_traps.py`
2. `scripts/eval_local_agent.py`
3. （新增）执行能力回归：命令生命周期/超时/审批场景集

### 4.3 验收阈值

- 逻辑陷阱：`loop_avg >= 0.95`
- 本地任务：`avg_score >= 85`
- 执行能力：关键场景通过率 `>= 95%`
- 运行异常：无未捕获异常；工具参数错误率显著下降

## 6. 风险与规避

- **模型服务不可用**：增加 preflight 检查，失败即中止并标记为环境失败而非功能失败
- **长任务阻塞**：执行与评估脚本分离，支持超时与中断恢复
- **策略漂移**：关键系统提示与模板纳入回归快照对比

---
